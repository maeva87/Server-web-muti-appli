# Server-web-muti-appli

Projet minimal pour le Sujet 4 — Serveur Web multi-applications.

Objectif (début): mettre en place une preuve de concept simple avec :
- un reverse proxy (nginx)
- au moins deux applications (ex. une app Node.js et une app statique/WordPress)
- accès HTTPS via certificats (démo avec certificat auto-signé)

Livrables initiaux :
- VM / conteneurs de démonstration
- configuration nginx pour sous-domaines
- documentation courte expliquant comment ajouter un site

Prochaines étapes :
1. Ajouter un `docker-compose.yml` minimal (nginx + app-node + app-wordpress)
2. Fournir commandes pour générer un certificat auto-signé et activer TLS
3. Rédiger le guide d'ajout d'un nouveau site

Voir la TODO list pour le suivi des tâches.
